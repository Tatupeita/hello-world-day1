## プロジェクトの目的
TAPPEI の実装練習 Day 1。Cursor → GitHub → Vercel の 9 ステップフロー
を身体化する。hair helpman 本体とは別の、独立した練習リポジトリ。

## 技術スタック
- HTML / CSS のみ(Day 1 時点)
- Day 2 以降は JavaScript、以後 Next.js へ段階移行予定

## ディレクトリ規約
- /index.html : トップページ(今日の唯一のファイル)
- /CLAUDE.md : このファイル(Cursor Agent 用プロジェクト設定)

## 実装ルール
- 変更は必ず Plan → Execute → Verify の順で行う
- 1 変更 = 1 コミット。コミットメッセージは「N 周目: 何をしたか」形式
- push は Cursor の Sync Changes で。ターミナルの git push は使わない
- 変更後は必ず Vercel URL を iPhone で再読込して反映を目視確認する

## 禁則
- hair helpman のブランド世界観(海×女神×波)は本練習で触らない。色は参考使用可
- Cursor Agent に指示する時は必ず「何をどのファイルにどう書くか」を Plan に書いてから投げる
